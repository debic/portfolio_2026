# 🖼️ Project Grid

A portfolio is often the first impression a developer makes, but first impressions shouldn't feel like a static resume.
This project is my personal portfolio and case-study hub, built with **React, TypeScript, and Vite** to showcase not just what I've built, but how I think and work.

I built this with a focus on **motion, craft, and clarity** — a site that feels alive without ever getting in the way of the content.

## ❤️ The Inspiration

This portfolio holds a special place for me: it's the first project where I worked hands-on with **Claude** as a real development partner, from the splash screen down to the routing logic.

That experience reshaped how I approach building interfaces, helping me move faster without losing control over the final product. I wanted this site to reflect that: technically solid, but with a very human point of view behind every animation, color choice, and layout decision.

My goal was simple: build something that feels like *me*, not a generic template, but a space where visitors can actually feel the care that went into every project I've shipped.

## 🛠️ Key Frontend Features

* **Animated Splash & Hero Reveal:** A custom GSAP-powered intro sequence and scroll-based hero reveal that sets the tone before a single word is read.
* **Project Grid:** No boxy, uniform squares here. Projects stagger across columns at their own height, creating an editorial, magazine-like rhythm that makes scrolling through my work feel like flipping through a lookbook instead of scanning a spreadsheet.
* **Context-Aware Navigation:** Project detail pages remember whether a visitor arrived from the home page or the Projects page, and the "Back" button adapts its destination and label automatically.
* **Responsive & Accessible:** From the mobile hamburger menu to the project grid, every section is built to hold up across devices.
